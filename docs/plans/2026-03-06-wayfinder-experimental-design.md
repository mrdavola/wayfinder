## Layer 1: Onboarding & Intake

### L1.1 Learner Engagement — Treasure Map Journey System
**Status:** [ ] Not started
**Theme:** Every project is an expedition, every learner is an explorer.
**Target feel:** Zelda: Wind Waker — approachable for ages 8-14, not patronizing.

#### 1a. XP & Progression System

**Core mechanic:** Learners earn **Explorer Points (EP)** for actions throughout their journey.

| Action | EP |
|---|---|
| Complete a stage | 50 |
| Submit quality work (AI-rated) | +10-30 bonus |
| Respond to Challenger | 25 |
| Write a reflection | 20 |
| Help a peer (social) | 15 |
| Complete a full project | 200 |
| Streak (3+ days active) | 10/day |

**Explorer Ranks:**
1. Apprentice (0 EP)
2. Scout (200 EP)
3. Pathfinder (600 EP)
4. Trailblazer (1500 EP)
5. Navigator (3000 EP)
6. Expedition Leader (6000 EP)

**Badges** — earned for specific achievements, displayed on profile:
- "First Expedition" — complete first project
- "Deep Diver" — write 5+ reflections
- "Devil's Advocate" — respond to 10 challenges
- "Cartographer" — help map a peer's journey
- "Streak Explorer" — 7-day streak

**Storage:** New `student_xp` and `student_badges` tables in Supabase. EP calculated from real actions, not hardcoded.

#### 1b. Treasure Map Visual Redesign

Replace the current SVG stage map with an illustrated treasure map canvas.

**Visual language:**
- Parchment background (uses existing `--parchment` CSS var)
- Hand-drawn style paths between stages (dotted trail lines)
- Each stage is a **landmark** — cave, lighthouse, bridge, volcano, camp, etc. (AI assigns thematic landmarks based on stage content)
- Current stage glows/pulses gently
- Completed stages show a planted flag with checkmark
- Locked stages are fog-covered (visible but mysterious)
- Unlock animation: fog rolls away, landmark fades in, sparkle burst

**Explorer token:** Small compass or avatar marker on current stage, animates along path when advancing.

**Implementation:** SVG-based (no canvas/WebGL), procedurally generated per-project. Each project gets a unique map shape based on number of stages + branch points.

#### 1c. Interactive Stage Types

Not every stage is "read and write." AI assigns stage types during generation.

| Stage Type | Experience | When Used |
|---|---|---|
| **Research Camp** (default) | Read prompt, submit written work | Knowledge gathering stages |
| **Puzzle Gate** | Drag-and-drop sorting, matching, or sequencing | Stages testing comprehension |
| **Choice Fork** | Choose-your-own-adventure branch — pick a path | Mid-project decision points |
| **Evidence Board** | Detective-style: given clue cards (facts, quotes, data), drag onto a corkboard to build an argument with connections | Stages requiring critical thinking, analysis, building a case |
| **Timed Challenge** | Countdown timer + quick-fire questions | Review/recall stages (FUTURE) |
| **Peer Exchange** | Submit work, review a peer's work | Collaborative stages (FUTURE) |

**For this build:** Research Camp (exists) + Puzzle Gate + Choice Fork + Evidence Board. Timed Challenge and Peer Exchange are future.

#### 1d. Narrative & Atmosphere

**Stage narration:** Each stage opens with 1-2 sentence narrative hook generated by AI:
> *"You've reached the Cartographer's Bridge. The river below carries fragments of data — before you cross, you'll need to piece together what you've learned..."*

**Sound (optional, toggleable):** Subtle ambient sounds per landmark type — campfire crackle, ocean waves, wind. Web Audio API or `<audio>` tags. Off by default, toggle in corner.

**Transitions:** Completing a stage triggers path-drawing animation to next landmark (1-2 sec, CSS/SVG animation).

#### 1e. Social & Team Elements

**Group projects:**
- Team members visible on map as separate explorer tokens
- Each member's progress visible ("Ava is at The Observatory, Marcus is at The Bridge")
- **Campfire chat** — simple group chat at each stage (stored in `guide_messages` with `group_chat` type)

**All projects:**
- **Explorer Log** — public feed on StudentHome showing recent completions (opt-in)
- Profile shows EP, rank, badges, completed expeditions count

#### Key Trade-offs
- SVG over Canvas/WebGL — simpler, works everywhere, matches codebase
- 2 interactive stage types now (Puzzle Gate + Choice Fork), rest later
- Sound is optional/toggleable
- EP values are tunable — easy to rebalance

## Layer 2: Architectural & Feature Shifts

### L2.1 Yearly Planning Mode
**Status:** [ ] Not started
**Priority:** HIGH — Barbara's top request

**Overview:** A dedicated "Year Plan" top-level page where guides map out a full year of projects per learner. AI generates suggestions, guide curates, plan adapts as the year progresses. Single project creation remains unchanged — this is an additional mode, not a replacement.

**How it works:**

1. **Setup:** Guide selects a learner (or group), AI ingests their profile (passions, interests, skill levels, parent outcomes) AND required standards/outcomes for the year.
2. **Generation:** AI produces ~15-20 project ideas. Each is a card with: title, 2-sentence description, target standards it would cover, estimated duration, interest alignment score.
3. **Curation:** Guide picks ~7 projects, drags them into sequence across the year. Can swap, remove, or request more suggestions. AI shows a "coverage meter" — which outcomes are covered vs gaps remaining.
4. **Living plan:** After each project completes, AI reassesses: "Here's what got covered, here's what's outstanding. Based on how this learner has grown, I'd suggest swapping Project #5 for this instead." Guide approves or dismisses.
5. **One-offs:** Guide can always create a single project outside the plan. If it covers standards, AI notices and adjusts the remaining plan.
6. **Build when ready:** Projects in the plan are placeholders (title + description + target standards). Guide clicks "Generate Now" when ready to run one — pre-fills QuestBuilder with the plan data.

**UI — Year Plan Page:**
- New top-level nav item: "Year Plan" in TopBar
- Learner selector at top (dropdown or tabs for each learner)
- View switcher (top-right corner, remembers preference):

| View | Best for |
|---|---|
| Journey Map (default) | Big picture, presentations, motivation |
| Timeline | Scheduling, sequence and pacing (horizontal Gantt-style) |
| List | Quick editing, drag between months (vertical, grouped by month/quarter) |
| Calendar | Spotting density, balancing workload across weeks |

- All views show same data, different layout
- Coverage meter sidebar: visual bar showing % of outcomes covered by selected projects
- "Suggest More" button to get additional AI ideas
- Each project card is draggable, clickable for detail, has "Generate Now" and "Remove" actions

**Storage:** New `year_plans` and `year_plan_items` tables in Supabase.
- year_plans: id, guide_id, student_id, school_id, school_year, created_at
- year_plan_items: id, plan_id, position, title, description, target_standards (JSONB), estimated_weeks, interest_tags, quest_id (nullable — linked once generated), status (planned/active/completed/swapped)

**AI functions needed:**
- ai.generateYearPlan(studentProfile, outcomes, existingCoverage) → returns ~15-20 suggestions
- ai.reassessYearPlan(planId, completedQuests, remainingOutcomes) → returns swap suggestions

### L2.2 Real-World Problem Integration
**Status:** [ ] Not started
**Priority:** HIGH

**Overview:** AI uses web search to ground projects in current, real-world problems and stakeholders. Available in both Year Plan generation and single project creation (QuestBuilder).

**How it works:**

1. **During generation:** AI searches the web for current problems relevant to the topic/standards/learner interests. Finds real stakeholders, organizations, data, and news.
2. **Auto-woven:** Real-world context is woven directly into generated project content (stage descriptions, guiding questions, deliverables) — not presented as a separate step.
3. **Transparent citations:** Each real-world reference shows inline with a small citation card:
   - **Green checkmark** — .gov, .edu, major news outlets (auto-trusted)
   - **Yellow flag** — blogs, lesser-known sites ("review suggested")
   - Citation shows: source name, URL, date accessed
4. **Guide review in Step 5:** During QuestBuilder review, guide sees all citations. Can click to view full source, edit the framing, or remove. Not a blocking gate — just transparent.
5. **Tagging:** Projects auto-tagged "Verified Real World" (has trusted sources) vs "Inspired by Real Issues" (AI-generated scenario based on real themes). Tagging system already exists.

**Technical approach:**
- Use Anthropic's tool_use with a web search tool (or integrate a search API like Tavily/Serper)
- AI receives search results as context, selects relevant ones, weaves into content
- Citations stored in quest_stages as a `sources` JSONB array: [{title, url, domain, trust_level, accessed_at}]

**AI functions needed:**
- Extend ai.generateQuest() to accept a `useRealWorld: true` flag
- New internal helper: searchRealWorldProblems(topic, standards, interests) → returns sourced problems

### L2.3 Optional Career Pathways Module
**Status:** [ ] Not started
**Priority:** MEDIUM

**Overview:** A dedicated exploration space where learners can discover career connections to their projects and interests. Separate from the existing career pathway step in QuestBuilder.

**How it works:**

1. **Contextual insights in projects:** When a project touches real-world problems, AI naturally surfaces career connections: "This project about water filtration connects to Environmental Engineering, Public Health, and Urban Planning careers." Shown as a small expandable card within stages.
2. **Dedicated Career Explorer page:** New page accessible from learner profile or StudentHome.
   - Shows careers the AI has connected to the learner's completed projects
   - Each career card: title, 1-sentence description, "Why this fits you" (based on their interests/skills), related projects they've done
   - "Explore More" button: AI suggests careers they haven't encountered yet based on their profile
   - Links to real resources (Bureau of Labor Statistics, career videos, etc.) — uses the same web search + trust system from L2.2
3. **Growth over time:** As learners complete more projects, their career map grows. Patterns emerge: "You keep gravitating toward design + technology — have you considered UX Design?"

**UI:**
- Career Explorer page: `/careers/:studentId` (or tab within student profile)
- In-project career cards: small expandable section in stage view
- No career content shown unless the learner or guide opts in (toggle in settings)

**Storage:** Extend existing `career_pathway` field. New `student_career_insights` table: id, student_id, career_title, description, reason, related_quest_ids (JSONB), source_urls (JSONB), created_at.

### L2.5 "Truth Protocol" (bundled into all AI output)
**Status:** [ ] Not started
**Priority:** HIGH — infrastructure layer

**Overview:** Not a separate feature or page. This is a verification layer baked into ALL AI output across the platform.

**How it works:**

1. **Source trust tiers:**
   - **Tier 1 (green):** .gov, .edu, major news (AP, Reuters, NYT, BBC), peer-reviewed journals → auto-trusted
   - **Tier 2 (yellow):** Known organizations, established blogs, Wikipedia → "review suggested"
   - **Tier 3 (red):** Unknown sources, personal blogs, social media → flagged, requires guide approval
2. **Hallucination guardrails:**
   - AI prompted to say "I'm not sure about this" rather than fabricate
   - When AI makes factual claims in project content, it must cite a source or explicitly mark as "AI-generated scenario"
   - Master prompt (P2) includes instruction: "Never present unverified facts as truth. If you cannot cite a source, frame it as a question or hypothesis."
3. **Visual indicators everywhere:**
   - Project content: inline citation badges (same as L2.2)
   - Field Guide chat: factual claims show a small "i" icon linking to source
   - AI-generated text without sources shows a subtle "AI-generated" watermark
4. **Guide override:** Guides can mark any AI claim as "verified by guide" (adds a blue badge) or "incorrect" (removes it with a note)

**Implementation:**
- Extend the WAYFINDER_SYSTEM_PROMPT (P2) with truth protocol instructions
- Add `sources` JSONB field to any table that stores AI-generated content (quest_stages already planned, extend to guide_messages, submission_feedback)
- TrustBadge component: small reusable UI component showing green/yellow/red badge with tooltip
- No external verification API needed initially — trust tier is determined by domain matching against a curated allowlist

### L3.1 Hands-on vs Digital Toggle
**Status:** [ ] Not started
**Priority:** LOW — simple addition

**Overview:** A toggle in QuestBuilder that tells the AI whether the project should emphasize physical/hands-on activities or digital/screen-based work.

**How it works:**
1. **QuestBuilder Step 1** (or Step 2): Toggle switch or segmented control with three options:
   - **Hands-on** — physical experiments, building, art, fieldwork, interviews
   - **Digital** — research, writing, coding, digital design, presentations
   - **Mixed** (default) — AI decides per stage what makes sense
2. **AI adapts:** The selection is passed to ai.generateQuest() as a `projectMode` parameter. AI adjusts:
   - Deliverables: "Build a physical model" vs "Create a digital prototype"
   - Materials lists: physical supplies vs software/tools

3. **Guide can override per-stage** during review (Step 5) — the toggle sets the default, not a hard rule.

**Implementation:**
- Add `project_mode` field to quests table (enum: 'hands_on', 'digital', 'mixed', default 'mixed')
- UI: segmented control component in QuestBuilder
- Extend ai.generateQuest() prompt to respect project mode

### L3.2 Hybrid Learning Components
**Status:** [ ] Not started
**Priority:** HIGH — core data capture

**Design principle: Never feel like school, but capture school-quality data behind the scenes.**

**Overview:** Academic skill assessment woven invisibly into the adventure experience. Three assessment modes used at different moments:

#### Mode 1: Quick Knowledge Checks
- Short, low-friction checks at the start of a stage
- NOT multiple choice — instead, framed as expedition challenges:
  - "Before you cross the bridge, quick — estimate how many planks you'd need if each is 2.5 feet and the gap is 15 feet"
  - "The map fragment is torn. Fill in the missing number in this pattern: 2, 6, 18, __, 162"
- Feels like a puzzle, captures math/reading comprehension data
- Auto-assessed by AI, result stored silently

#### Mode 2: Applied Problems in Deliverables
- The stage deliverable itself IS the assessment
- "Calculate the budget for your community garden using the price list provided" — tests multiplication, addition, budgeting
- "Write a persuasive letter to the city council about your water filtration findings" — tests writing, argumentation
- AI evaluates the submission for both project quality AND academic skill demonstration
- Guide sees: "This submission demonstrated: multiplication (strong), persuasive writing (developing), data interpretation (strong)"

#### Mode 3: Skill Check Conversations
- Field Guide naturally probes understanding during chat:
  - "Interesting approach! Can you walk me through why you divided by 3 there?"
  - "What would happen if you doubled the amount? How would that change your result?"
- AI assesses depth of understanding from the conversation (not a quiz — a dialogue)
- Integrated into existing Field Guide chat, no separate UI needed

#### How results are surfaced:

**Learner view (progress, never grades):**
- In their profile/XP section: "Skills demonstrated" list with warm language
- "You've shown strong fraction skills across 3 projects"
- "Your persuasive writing is growing — keep it up"
- Ties into EP system: demonstrating a skill = bonus EP
- NO percentages, NO letter grades, NO red/fail indicators

**Guide view (detailed, actionable):**
- Per-learner skill breakdown: each standard with status (emerging/developing/strong/mastered)
- "Ava struggles with fraction division but nails multiplication"
- "Marcus hasn't encountered persuasive writing yet — consider adding it to his next project"
- Feeds into Year Plan (L2.1) — AI uses this data to suggest projects targeting gaps
- Exportable for parent meetings or reporting requirements

**Storage:**
- Extend existing `student_skills` table with assessment data: assessment_type (check/applied/conversation), evidence_text, assessed_at
- New `skill_assessments` table: id, student_id, skill_id, quest_id, stage_id, assessment_type, ai_rating (1-4 scale internally), evidence, created_at
- Guide dashboard queries aggregate these into the readable breakdown

**AI functions needed:**
- Extend ai.reviewSubmission() to return skills_assessed with ratings
- Extend Field Guide prompt to probe for understanding and log assessments
- New: ai.generateKnowledgeCheck(stageContent, targetSkills) → returns adventure-framed check question

## Layer 4: Progress, Mastery & Community

### L4.1 Visual Progress Tracking
**Status:** [ ] Not started
**Priority:** HIGH — makes mastery visible and motivating

**Overview:** A "Mastery Map" page showing all skills and standards a learner has encountered across all projects. Two switchable views: World Map (treasure map extension) and Knowledge Graph (Obsidian-style).

**View 1: World Map (default)**
- Zoomed-out version of the treasure map metaphor
- Each completed project is an **island or region** on a larger world map
- Skills are **landmarks discovered** across all expeditions — a lighthouse for "persuasive writing," a mountain for "fractions," a port for "data analysis"
- Uncharted territory (greyed/fogged areas) = skill categories not yet explored
- Islands grow in detail as more projects are completed in that area
- Clicking a landmark shows: skill name, mastery level, which projects touched it, recent evidence
- Color intensity shows mastery: faint outline (emerging) → solid color (developing) → glowing (mastered)
- Paths between islands show connections: "Your ocean research project connected Science and Writing"

**View 2: Knowledge Graph (Obsidian-style)**
- Nodes = skills/standards, sized by mastery level
- Edges = connections (two skills touched in the same project get linked)
- Clusters form naturally: math skills clump together, science skills clump, with bridges between
- Interactive: drag nodes, zoom, click for detail
- Color by category (math = blue, science = green, writing = gold, etc.)
- Unconnected/small nodes at the periphery = gaps to address
- Implementation: SVG with a simple force-directed layout (d3-force or a lightweight alternative)

**Shared features (both views):**
- Toggle switch in top-right: "World Map | Knowledge Graph"
- Time slider: drag to see how the map looked 1 month ago, 3 months ago — watch it grow
- Summary stats bar: total skills encountered, % mastered, strongest area, biggest gap
- Accessible from: learner profile, guide's student detail page, parent dashboard

**Who sees what:**
- Learner: World Map by default, can switch to Knowledge Graph. Framed as exploration progress, not grades
- Guide: Both views + detailed breakdown table underneath. Can export for reports
- Parent: World Map only (via parent dashboard). Simple, visual, encouraging

**Storage:** No new tables needed. Queries from existing `student_skills`, `skill_assessments`, `skill_snapshots`. The visualization is a frontend concern — all data already exists.

**Technical approach:**
- SVG-based for World Map (consistent with Layer 1 treasure maps)
- d3-force (or similar lightweight lib) for Knowledge Graph node layout
- No Pinecone or vector DB needed — all structured relational queries from PostgreSQL/Supabase
- New page: `/progress/:studentId`

### L4.3 Accountability Buddy Layer
**Status:** [ ] Not started
**Priority:** MEDIUM

**Overview:** Multi-layer accountability system with peer encouragement, guide awareness, and optional parent flagging.

**How it works:**

1. **Buddy pairing:**
   - Guide pairs learners as accountability buddies (manual, in Students page or Group Builder)
   - Or learner picks their own buddy from their school
   - Each learner has one buddy (can be changed anytime)

2. **Stall detection:**
   - System tracks last activity per learner (stage completion, submission, note, chat)
   - "Stall" = no activity for 3+ days (configurable by guide)

3. **Notification layers:**

| Who | Trigger | How | Automatic? |
|---|---|---|---|
| **Buddy** | 3+ days inactive | In-app nudge: "Ava hasn't been on her expedition lately — want to send an encouraging message?" | Yes |
| **Guide** | 3+ days inactive | Dashboard alert: yellow indicator on student card, "Hasn't been active in 4 days" | Yes |
| **Parent** | Guide decides | Guide clicks "Flag to Parent" button on student card → shows on parent dashboard as "Guide noted: hasn't been active this week" | Manual only — guide must click |

4. **Buddy messaging:**
   - Simple message system between buddies (not full chat — one encouragement message)
   - Pre-written templates available: "Hey, your expedition misses you!" / "Want to work on our projects together?"
   - Custom message option with guide-visible log (safety)

**Storage:**
- New `buddy_pairs` table: id, student_a_id, student_b_id, school_id, created_at
- New `buddy_messages` table: id, pair_id, sender_id, message, created_at
- New `activity_log` table (or extend existing): student_id, action_type, timestamp — for stall detection
- `parent_access` table extended with `guide_flags` JSONB array for manual alerts

### L4.4 Community Repository
**Status:** [ ] Not started
**Priority:** MEDIUM

**Overview:** School-scoped library where guides and learners share completed projects. Others can browse, rate, and remix. Built to expand cross-school later.

**How it works:**

1. **Sharing a project:**
   - When a project is completed, guide (or learner with guide approval) can click "Share to Community"
   - Adds a snapshot of the project to the community repository (not a live link — a frozen copy)
   - Guide adds tags: subject area, grade band, duration, project mode (hands-on/digital/mixed)

2. **Browsing:**
   - New page: `/community` (or section within existing Library page)
   - Filterable grid (similar to QuestLibrary): by subject, grade, rating, duration
   - Search by keyword
   - Each card shows: title, description preview, star rating, author school, tags, "Used X times"

3. **Rating & reviews:**
   - Guides can rate (1-5 stars) and leave short reviews
   - "This worked great for my 10-year-olds"
   - "Stage 3 was the highlight — my students loved the hands-on part"
   - Average rating shown on card

4. **Clone (guides):**
   - "Use as Template" button → clones into QuestBuilder pre-filled
   - Guide can customize before generating — it's a starting point, not locked

5. **Remix (learners):**
   - "Remix this Project" button on completed projects
   - Learner forks the project, modifies interests/topic/approach
   - Runs through a simplified builder: "What would you change about this project?"
   - AI regenerates stages based on the remix while keeping the core standards
   - **Requires guide approval** before becoming active
   - Original project credited: "Remixed from Ava's Ocean Ecosystem project"

**Scope: School-only for now.**
- Community page only shows projects from your school
- Data model includes school_id so cross-school is a config change, not a rebuild

**Storage:**
- New `community_projects` table: id, quest_id (source), school_id, shared_by (guide/student id), title, description, tags (JSONB), avg_rating, use_count, created_at
- New `community_reviews` table: id, community_project_id, reviewer_id, rating (1-5), review_text, created_at
- New `community_remixes` table: id, source_community_project_id, remixed_by (student_id), new_quest_id, status (pending_approval/approved/rejected), created_at

## Layer 5: "Eventually" & Future State

### L5.A — Year Plan v2 (merges L5.1 + L5.2)
**Status:** [ ] Future — depends on L2.1 (Year Plan) being built and tested
**Priority:** Evolve after Year Plan v1 is in use

**Direction:**

This is not a new feature — it's the Year Plan (L2.1) getting smarter.

**Outcome Balancing (from L5.1):**
- AI doesn't just suggest projects — it ensures the "Bank of Skills" (all required outcomes) is spread evenly across the ~7 selected projects
- Visual coverage matrix: rows = outcomes, columns = projects. Each cell shows which project hits which outcome. AI highlights gaps and overloads
- "Rebalance" button: AI suggests swaps or additions to fill gaps without overloading any single project
- Guide always has final say — AI suggests, guide approves

**Dual Creator Modes (from L5.2):**
- Year Plan page gets a mode toggle at the top:
  - **Single Project** — current QuestBuilder flow, unchanged
  - **Year Planner** — the full Year Plan experience (L2.1) with outcome balancing layered on
- These aren't separate codepaths — Year Planner IS the Year Plan page. Single Project IS QuestBuilder. The "mode" is just which entry point the guide uses.
- A "Package" export: guide can export a balanced 7-project plan as a template that other guides can import via Community Repository (L4.4)

**Dependencies:** L2.1 (Year Plan), L3.2 (skill assessments feeding gap data), L4.4 (community sharing)

### L5.B — Game World (merges L5.3 + L5.4)
**Status:** [ ] Future — depends on Layers 1-4 being solid
**Priority:** Long-term vision, build incrementally

**Direction:**

The evolution from flat stage map → treasure map (L1) → world map (L4.1) → fully explorable game world.

**Phase 1 — Branching Narratives (from L5.3):**
- Extend Choice Fork (L1.1c) from single decision points to full branching storylines
- A project isn't linear anymore — it's a narrative tree. Different choices lead to different stages, different challenges, different outcomes
- AI generates the full tree during project creation (not just a linear sequence with one fork)
- Learner's path through the tree is unique — two students doing the "same" project may have very different experiences
- Still SVG-based, still within current architecture

**Phase 2 — Persistent World (from L5.4):**
- The World Map (L4.1) evolves from a read-only visualization into an explorable space
- Learner has an avatar that moves through a persistent world
- Completed projects become permanent regions they can revisit
- New projects appear as unexplored territory on the horizon
- Other learners' avatars visible (social presence)
- **This is where Three.js comes in** — 2.5D or 3D explorable world
- Skills are embedded in the world as discoverable objects, landmarks, NPCs
- Mastery unlocks new areas, abilities, or tools within the world

**Phase 3 — World Building:**
- Learners don't just explore — they build
- Completing a project lets you add a structure/landmark to your world
- Your world becomes a portfolio: visitors can see what you've built and learned
- Guides can "visit" a learner's world to see their progress spatially

**Tech notes:**
